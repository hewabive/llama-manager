# Документация аргументов llama-server

Эта папка является источником подробной инженерной справки по аргументам
`llama-server`. Документы предназначены для UI llama-manager и для работы
кодинг-агентов, которые будут поддерживать справку после обновлений llama.cpp.

Правило хранения: один аргумент - один Markdown-файл. Имя файла строится из
основного имени аргумента без начальных дефисов:

- `--ctx-size` -> `ctx-size.md`
- `--n-gpu-layers` -> `n-gpu-layers.md`
- `--models-preset` -> `models-preset.md`

Приложение читает эти файлы во время работы и показывает статус документа в
разделе Arguments. Большой объем текста допустим: лучше держать подробности,
примеры, ссылки на issues/PR и локальные наблюдения рядом с конкретным
аргументом, чем смешивать все в общей базе знаний.

Статусы:

- `draft` - черновик, создан автоматически или еще не проверен по исходному коду.
- `current` - документ проверен для указанного `reviewedHelpHash` и commit llama.cpp.
- `needs-review` - справка `llama-server --help` изменилась после проверки.
- `deprecated` - аргумент устарел, но еще может встречаться у пользователей.
- `orphaned` - файл остался, а аргумента больше нет в текущем `--help`.

Требования к хорошему документу:

- основной текст на русском языке;
- краткое практическое резюме в `summary`;
- объяснение фактического поведения в llama.cpp;
- формат значения, значения по умолчанию, допустимые enum/list варианты;
- влияние на RAM, VRAM, latency, throughput и время старта, если применимо;
- взаимодействие с другими аргументами;
- типовые ошибки, симптомы в логах и способы диагностики;
- примеры команд и/или пресетов;
- ссылки на исходный код, issues, PR, discussions и локальные наблюдения.

Для массовой проверки и генерации черновиков используйте:

```bash
pnpm --filter @llama-manager/api args:docs:audit
pnpm --filter @llama-manager/api args:docs:audit -- --write
pnpm --filter @llama-manager/api args:docs:audit -- --write --rewrite-drafts
```

`--write` создает недостающие файлы по текущему `llama-server --help`, но не
делает документы финальными. После ручной или агентной проверки обновите
frontmatter, укажите commit llama.cpp и поставьте `docStatus: current`.

`--rewrite-drafts` перегенерирует только файлы со статусом `draft`. Документы
со статусом `current` не перезаписываются, чтобы не потерять ручную или
агентную инженерную работу.

# --log-prompts-dir

## Кратко

`--log-prompts-dir` включает дамп промптов: при непустом значении сервер на каждый входящий запрос пишет содержимое поля `prompt` в отдельный файл внутри указанного каталога. Значение хранится в `common_params::path_prompts_log_dir`. По умолчанию выключено (пустая строка). Аргумент задуман как отладочный.

## Оригинальная справка llama.cpp

```text
Log prompts to directory (only used for debugging, default: disabled)
```

## Паспорт аргумента

- Основное имя: `--log-prompts-dir`
- Категория в `--help`: `Параметры llama-server`
- Тип значения в llama-manager: `path`
- Подсказка формата из `--help`: `PATH`
- Переменная окружения: нет (`set_env` не задан)
- Поле в `common_params`: `path_prompts_log_dir`
- Значение по умолчанию: пусто (отключено)
- Примеры применения в коде: `LLAMA_EXAMPLE_SERVER`, `LLAMA_EXAMPLE_CLI`

## Что меняет в llama-server

Если `path_prompts_log_dir` непуст, при обработке каждого запроса (`server-context.cpp`) сервер формирует имя файла из миллисекундного таймстампа (`%012d.txt`) и записывает в него промпт: строку как есть либо `dump(2)` для структурированного prompt. Если файл создать не удалось, в лог идет `failed to create <path>`.

Каталог должен существовать заранее: код только создает файлы внутри него, сам каталог не создается. Каждый запрос — отдельный файл, ротации и очистки нет, так что на нагруженном сервере каталог быстро разрастается.

## Значения и формат

Путь к каталогу. Относительный путь считается относительно рабочего каталога процесса. Файлы получают имена вида `000123456789.txt` по времени запроса.

## Когда использовать

Только для отладки: воспроизвести, какой именно промпт дошел до модели (после применения шаблонов/прокси), разобрать проблему с конкретным запросом, собрать корпус промптов в контролируемой среде. Не включайте в проде — это и приватность промптов на диске, и неограниченный рост каталога.

## Влияние на производительность и память

Добавляет синхронную запись файла на каждый запрос — заметная нагрузка на диск при высоком RPS и потенциально безграничный объем данных. На сам инференс не влияет.

## Взаимодействие с другими аргументами

- `--log-file`: пишет общий лог сервера в один файл; `--log-prompts-dir` — отдельные файлы только с промптами.
- `--verbose`: повышает детализацию логов, но промпты по соображениям размера в основной лог не печатаются — для них и существует этот флаг.

## Типовые проблемы и диагностика

- В логах `failed to create <path>`: каталог не существует или нет прав на запись.
- Каталог не появился сам: создайте его заранее (`mkdir -p`).
- Каталог бесконтрольно растет: это ожидаемо, отключайте флаг после отладки.

## Примеры

```bash
mkdir -p /tmp/llama-prompts
llama-server --model /models/model.gguf --log-prompts-dir /tmp/llama-prompts
```

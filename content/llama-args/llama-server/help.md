# --help

## Кратко

`--help` печатает справку по аргументам и завершает процесс. Для `llama-manager` это полезно как диагностическая команда для сверки версии `llama.cpp`, но не как аргумент instance config: сервер с этим флагом не начнет слушать HTTP port.

## Оригинальная справка llama.cpp

```text
print usage and exit
```

## Паспорт аргумента

- Основное имя: `--help`
- Алиасы: `-h`, `--help`, `--usage`
- Категория в `--help`: `Общие параметры`
- Тип значения в llama-manager: `flag`
- Переменные окружения: нет
- Поле в `common_params`: `usage`
- Этап применения: парсинг CLI, до проверки обязательной модели и до старта server runtime

## Что меняет в llama-server

В `common/arg.cpp` обработчик ставит `params.usage = true`. После разбора аргументов `common_params_parse` вызывает `common_params_print_usage(ctx_arg)`, затем example-specific `print_usage`, если он передан, и вызывает `exit(0)`.

Проверка обязательного `--model` пропускается для help/completion paths, поэтому `llama-server --help` работает без модели.

В проверенной версии `llama.cpp` отдельный `--license` больше не входит в общий набор аргументов `llama-server`; для сверки доступных диагностических флагов ориентируйтесь на фактический вывод `--help` текущего бинарника.

## Значения и формат

Флаг не принимает значение. Формы `-h`, `--help` и `--usage` эквивалентны.

## Когда использовать

- Проверить фактический набор аргументов у установленного бинарника.
- Сверить изменения после обновления llama.cpp через source snapshot.
- Быстро увидеть env-переменные и default values из текущей сборки.

## Влияние на производительность и память

Модель не загружается, backend runtime сервера не стартует, HTTP порт не открывается. Память расходуется только на запуск процесса и вывод справки.

## Взаимодействие с другими аргументами

`--help` завершает процесс после разбора. Другие runtime-аргументы могут быть распознаны, но сервер не стартует. Не добавляйте `--help` в постоянный preset instance.

## INI-пресеты и router-режим

В `--models-preset` этот флаг не имеет практического смысла: дочерний процесс модели выведет usage и завершится вместо обслуживания запросов.

## Типовые проблемы и диагностика

- Сервис мгновенно завершился с кодом 0 и большим текстом usage: проверьте, не попал ли `--help` или `--usage` в argv.
- UI считает это list-аргументом: это ошибка metadata; по source это флаг без значения.

## Примеры

```bash
llama-server --help
```

```bash
llama-server --usage
```

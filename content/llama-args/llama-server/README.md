# Документация аргументов llama-server

Эта папка является источником подробной инженерной справки по аргументам `llama-server`. Документы предназначены для UI llama-manager и для работы кодинг-агентов, которые будут поддерживать справку после обновлений llama.cpp.

Правило хранения: один аргумент - один Markdown-файл. Имя файла строится из основного имени аргумента без начальных дефисов:

- `--ctx-size` -> `ctx-size.md`
- `--n-gpu-layers` -> `n-gpu-layers.md`
- `--models-preset` -> `models-preset.md`

Приложение читает эти файлы во время работы и показывает их содержимое в разделе Arguments. Большой объем текста допустим: лучше держать подробности, примеры, ссылки на issues/PR и локальные наблюдения рядом с конкретным аргументом, чем смешивать все в общей базе знаний.

Отдельных статусов у документов нет. Если аргумент исчез из актуального generated help-блока llama.cpp, соответствующий Markdown-файл удаляется из репозитория после проверки, что аргумент не был просто переименован.

Требования к хорошему документу:

- основной текст на русском языке;
- краткое практическое резюме в `summary`;
- объяснение фактического поведения в llama.cpp;
- формат значения, значения по умолчанию, допустимые enum/list варианты;
- влияние на RAM, VRAM, latency, throughput и время старта, если применимо;
- взаимодействие с другими аргументами;
- типовые ошибки, симптомы в логах и способы диагностики;
- примеры команд и/или пресетов;
- ссылки на исходный код, issues, PR, discussions и локальные наблюдения.

Для синхронизации с llama.cpp используйте:

```bash
pnpm --filter @llama-manager/api args:docs:source-sync -- --diff
pnpm --filter @llama-manager/api args:docs:source-sync -- --write
pnpm --filter @llama-manager/api args:docs:quality -- --changed --strict
```

`source-sync -- --diff` показывает изменения generated help-блока. Новые документы создаются агентом по `_template.md` и соседним справочным файлам, а документы по удаленным аргументам удаляются после проверки, что это не rename.

## Агентная обработка

Для синхронизации после обновлений llama.cpp используйте repo-local Codex skill `.codex/skills/llama-arg-help-sync/SKILL.md`. Он опирается на diff между сохраненным generated help snapshot и текущим блоком `tools/server/README.md`.

Для подробного наполнения документов можно также использовать общий промпт `_agent-prompt.md`. Он фиксирует источники, границы редактирования и критерии качества, чтобы несколько кодинг-агентов могли параллельно обновлять разные группы аргументов без конфликтов.

Практический порядок работы:

- запустить `args:docs:source-sync -- --diff` после обновления llama.cpp;
- по diff определить затронутые аргументы;
- удалить документы по аргументам, которые исчезли из generated help-блока и не являются переименованными;
- при большой пачке разбить Markdown-файлы на непересекающиеся тематические группы;
- каждому агенту выдать только его список файлов и ссылку на skill или `_agent-prompt.md`;
- после обновления справки выполнить `args:docs:source-sync -- --write`;
- после завершения проверить, что в измененных файлах не осталось шаблонных фраз, `TODO` и неподтвержденных утверждений через `args:docs:quality`;
- снова запустить `args:docs:source-sync`, общий check проекта и только после этого коммитить.

# --sse-ping-interval

## Кратко

`--sse-ping-interval` задает, как часто сервер шлет keep-alive «ping» в SSE-поток, если за это время не появилось новых данных. Значение записывается в `common_params::sse_ping_interval` (секунды), по умолчанию `30`. Это спасает стриминговые запросы (`stream: true`) от обрыва клиентами, которые сами разрывают idle-соединение.

## Оригинальная справка llama.cpp

```text
server SSE ping interval in seconds (-1 = disabled, default: 30)
```

## Паспорт аргумента

- Основное имя: `--sse-ping-interval`
- Категория в `--help`: `Параметры llama-server`
- Тип значения в llama-manager: `number`
- Подсказка формата из `--help`: `N`
- Переменная окружения: `LLAMA_ARG_SSE_PING_INTERVAL`
- Поле в `common_params`: `sse_ping_interval`
- Значение по умолчанию: `30`
- Специальное значение: `-1` (и любое `<= 0`) отключает пинги

## Что меняет в llama-server

В цикле отдачи стрима (`server-context.cpp`) при ожидании следующего результата сервер замеряет время. Если `sse_ping_interval > 0` и с момента старта ожидания прошло больше `sse_ping_interval` секунд без данных, срабатывает таймаут ожидания и в поток отправляется SSE-ping, после чего ожидание продолжается. Так соединение остается «живым», даже если генерация задерживается (длинный prefill, очередь слотов, медленная модель).

Комментарий в коде прямо называет причину: некоторые клиенты (например, `undici`) рвут соединение, если данные не приходят какое-то время.

## Значения и формат

Целое число секунд. `-1` (или `0`) отключает пинги полностью. Слишком большой интервал может не успеть до клиентского idle-таймаута; слишком маленький создает лишний служебный трафик в потоке.

## Когда использовать

Уменьшайте интервал, если клиенты с агрессивным idle-таймаутом обрывают долгие стримы во время длинного prefill или ожидания свободного слота. Отключайте (`-1`) только если пинги мешают клиенту, который не умеет их игнорировать.

## Влияние на производительность и память

На инференс не влияет. Эффект — лишь периодические служебные SSE-кадры в активных стримах.

## Взаимодействие с другими аргументами

- `--timeout`: общий read/write таймаут HTTP; `--sse-ping-interval` работает внутри него и не дает соединению простаивать.
- `--threads-http`: число HTTP-потоков, обслуживающих в том числе стримы.

## Типовые проблемы и диагностика

- Клиент обрывает долгий стрим с network/idle-ошибкой: уменьшите `--sse-ping-interval`.
- В debug-логах видно `sending SSE ping`: пинг сработал, это норма.
- Лишние пустые события у нестандартного клиента: при необходимости отключите через `-1`.

## Примеры

```bash
llama-server --model /models/model.gguf --sse-ping-interval 10
llama-server --model /models/model.gguf --sse-ping-interval -1
```
